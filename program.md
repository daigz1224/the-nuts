# 项目规格书：单人德州扑克概率训练器 (The Nuts)

## 1. 项目概述

本项目是一个纯静态网页端的单机德州扑克训练软件。核心目的不是博彩娱乐，而是一款**概率计算与决策逻辑的教学工具**。

- **游戏类型：** 无限注德州扑克 (No-Limit Texas Hold'em)，6人桌 (6-max)，玩家 + 5个AI对手。
- **界面语言：** 中文 (zh-CN)。
- **架构要求：** 纯前端实现（Client-side only），无后端，无数据库，无网络请求。所有逻辑均在浏览器通过 JavaScript 运行。
- **部署目标：** 静态文件托管（Vercel、GitHub Pages）。

## 2. 设计语言与视觉风格

**核心基调：克制、精密、优雅。**

彻底抛弃传统棋牌游戏花哨的霓虹灯、廉价的金币特效和刻板的"发牌员"形象。视觉风格参考现代生产力工具（侧边栏布局、磨砂玻璃质感）以及硬核科幻电影的精密仪表盘，做到信息层级分明，视觉上不喧宾夺主。

### 2.1 配色方案

- 深色模式为主，背景使用深邃的太空灰（`#121212` 或 `#1C1E26`）。
- 扑克桌不用高饱和草绿色，改用碳灰色面板或深莫兰迪色系毛毡纹理。
- 交互高亮色：弃牌（柔和珊瑚红）、跟注（沉稳天青蓝）、加注（克制琥珀黄）。

### 2.2 排版与字体

- 数字、概率、底池金额使用等宽字体（`Roboto Mono` 或 `JetBrains Mono`）。
- 界面文本使用无衬线字体（`Inter`、`San Francisco`、`PingFang SC`、`Noto Sans SC`）。

### 2.3 布局

采用**可折叠侧边栏 + 主视区**布局，牌桌始终居中：

- **左侧边栏（可折叠）：** 概率知识库——Outs 图表、起手牌范围速查、常用概率速记。
- **中央主视区：** 牌桌——极简的玩家位置、筹码量、公共牌和底池。UI 元素在需要时浮现，不需要时隐藏。
- **右侧边栏/底部控制台（可折叠）：** 实时 HUD 数据面板（胜率、底池赔率）与玩家操作区。
- **响应式策略：** 平板/小屏下侧边栏折叠为底部抽屉或标签页切换；设定最小视口宽度 1024px 以保证核心体验。

### 2.4 动效设计

拒绝夸张特效。发牌、筹码移动采用流畅、符合物理直觉的平滑过渡（Easing 函数），注重交互反馈的干脆利落。

## 3. 技术栈

- **前端框架：** React + TypeScript
- **样式：** Tailwind CSS
- **动效：** Framer Motion
- **状态管理：** Zustand（管理牌局流转与手牌历史回放）
- **核心扑克逻辑：** 自研手牌评估器（不依赖 `pokersolver`）——自研可以让教学模块透明地向用户展示牌力计算过程。
- **胜率计算：** Monte Carlo 模拟——随机发出剩余公共牌 N 次，统计胜率。可配置采样次数以平衡速度与精度（默认 5000 次）。

## 4. 核心功能模块

### 4.1 游戏状态机引擎

- 管理完整的德扑生命周期：发牌前 → 翻前 (Pre-flop) → 翻牌 (Flop) → 转牌 (Turn) → 河牌 (River) → 结算 (Showdown)。
- 底池分割与筹码管理逻辑（包括 Side Pot）。
- 位置轮转：庄位 (BTN)、小盲 (SB)、大盲 (BB)、枪口位 (UTG) 等，每手自动轮转。

### 4.2 自研手牌评估器

不引入第三方比牌库，自行实现手牌评估：
- 支持所有标准牌型（皇家同花顺 → 高牌）的识别与排序。
- 评估过程可被教学模块调用，向用户展示"为什么这手牌是两对而不是顺子"等细节。
- 支持从 7 张牌（2 手牌 + 5 公共牌）中选出最优 5 张组合。

### 4.3 Monte Carlo 胜率引擎

- 给定已知手牌和公共牌，随机模拟剩余牌面，统计玩家胜率/平局率。
- 默认采样 5000 次，可由用户调节（精度 vs 速度）。
- 输出：胜率百分比、各牌型命中概率。

### 4.4 带位置感知的规则 AI

无需深度学习。AI 基于参数配置初始化。

**AI 参数结构：**
- VPIP（入池率）、PFR（翻前加注率）、AF（激进指数）、诈唬倾向 (Bluff Frequency)。
- **位置修正：** 同一个 AI 在不同位置（UTG vs BTN）会自动调整其入池范围和激进度。枪口位收紧，庄位放宽。

**决策树：**
- 翻前：查询本地预设的起手牌范围表 (Range Charts)，结合当前位置和随机数决策。
- 翻后：评估"当前牌力 + 听牌潜力（Outs）+ 底池赔率 + 位置优势"，通过加权打分得出行动（弃牌/跟注/加注）。

**3 种基础风格：**
- 紧凶型 (TAG)：入池少，入池就加注，位置感知强。
- 松弱型 (Passive Fish)：入池多，很少加注，容易被诈唬。
- 疯狂型 (Maniac)：高频加注，高频诈唬，难以预测。

### 4.5 教学与测验系统

**实时 HUD（可开关）：**
- 当前手牌胜率（来自 Monte Carlo 引擎）。
- Outs 数量与成牌概率。
- 底池赔率 (Pot Odds) 与所需胜率。
- 隐含赔率提示 (Implied Odds)。

**测验触发场景（不仅限于大注）：**
- **翻前：** "这手牌在当前位置是否应该入池？"
- **每条街：** "你有多少 Outs？当前底池赔率是多少？跟注是否正 EV？"
- **隐含赔率：** "底池赔率不足，但考虑隐含赔率，这个跟注是否合理？"
- **弃牌权益：** "半诈唬加注在这里是否正 EV？计算弃牌权益。"

**极简复盘：**
- 牌局结束后，非正 EV 的决策被高亮标记，附带简短的数学解释。
- 可回看完整手牌历史。

### 4.6 难度进阶系统

- **新手模式：** HUD 常驻，测验频繁，AI 行为可预测，每步决策后给出概率提示。
- **进阶模式：** HUD 需手动开启，测验在关键节点触发，AI 混合策略。
- **高手模式：** 无 HUD，无测验，AI 具备完整位置感知与混合策略，接近真实牌局。

### 4.7 本地数据持久化 (localStorage)

- 累计统计：胜率、测验正确率、常见错误类型。
- 手牌历史记录（供复盘回看）。
- 用户设置与偏好（难度、HUD 开关、采样次数）。
- 当前进度与难度等级。

## 5. 开发阶段划分

### Phase 1：核心逻辑与基础渲染

- 搭建项目骨架（React + TypeScript + Tailwind CSS），**同时配置好设计主题**（颜色、字体、间距 Token），从第一天就使用正确的视觉基础。
- 实现扑克牌生成、洗牌、发牌逻辑。
- 自研手牌评估器，支持所有标准牌型。
- 渲染基础牌桌，走通从翻前到结算的完整流程（先用简单的固定 AI 或随机 AI 验证流程）。

### Phase 2：AI 模块与完整对局

- 实现带位置感知的 AI 决策树。
- 实现 3 种 AI 风格（紧凶、松弱、疯狂）。
- 完善 6 人桌的完整对局流程，包括筹码流转、Side Pot、位置轮转。

### Phase 3：概率引擎与教学系统

- 实现 Monte Carlo 胜率模拟引擎。
- 实现实时 HUD 数据面板（胜率、Outs、底池赔率）。
- 实现多场景测验系统（翻前选择、Outs 计算、底池赔率、隐含赔率、弃牌权益）。
- 实现牌局复盘与手牌历史。
- 实现难度进阶系统。
- 接入 localStorage 持久化。

### Phase 4：UI/UX 深度打磨

- 精细化深色主题、毛玻璃效果、阴影层级。
- 增加发牌、筹码滑动的克制动效（Framer Motion）。
- 侧边栏折叠交互与响应式适配。
- 整体视觉走查，确保"高级感"与"克制感"的统一。
